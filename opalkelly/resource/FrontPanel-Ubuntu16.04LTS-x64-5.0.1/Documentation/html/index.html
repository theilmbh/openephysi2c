Overview

The FrontPanel API provides a powerful C++ interface to Opal Kelly USB and PCI Express FPGA boards.

To build an application using this API, you simply need to include okFrontPanelDLL.h from your code and ensure that okFrontPanel.dll can be found during your program execution, e.g. by putting it in the same directory as your executable.

To use the library, you need to create an instance of okCFrontPanel which encapsulates communication with the USB driver and provides access to FrontPanel endpoints. The simplest way to do it is to create an instance of OpalKelly::FrontPanelDevices object associated with the given "realm", by default just the realm of locally connected devices is used. You can then use its Open method to open the first, or all, of the available devices.

Note
Existing users of the library may continue to call OpenBySerial to open the device, however they are encouraged to update their code to use OpalKelly::FrontPanelDevices instead, please see the Switching to OpalKelly::FrontPanelDevices section of the documentation for more details.

Once you have an okCFrontPanel object, ConfigureFPGA is used to download a configuration file to the FPGA. Sending data to WireIn endpoints is done with calls to SetWireInValue followed by a call to UpdateWireIns to simultaneously update all Wire Ins. And ActivateTriggerIn is used to trigger the XEM.

The API uses polling to query the values of output endpoints such as WireOuts and TriggerOuts. The methods UpdateWireOuts and UpdateTriggerOuts perform these queries. Once the wire and trigger values have been updated by these methods, you can retrieve the values using the methods GetWireOutValue and IsTriggered or GetTriggerOutVector .
